# GitHub Project Management

Use GitHub Issues as the main planning system for this workspace.

## Workflow

1. Capture rough ideas as `Tool idea` issues.
2. Move promising ideas into a category folder when prototyping starts.
3. Track implementation details as `Tool task` issues.
4. Use pull requests for meaningful changes, even when working alone.
5. Move inactive tools to `Archive/` and close related issues.

## Labels

Recommended labels:

- `idea`: New tool concept.
- `task`: Implementation work.
- `bug`: Something broken.
- `docs`: README or usage notes.
- `health`: Health tools.
- `life`: Life tools.
- `knowledge`: Learning, reading, and notes tools.
- `finance`: Finance tools.
- `prototype`: Early version.
- `active`: Useful and maintained.
- `paused`: Temporarily stopped.
- `archived`: No longer active.

## Suggested GitHub Project Views

- `Backlog`: Open ideas and tasks.
- `Now`: Work currently being prototyped.
- `Active Tools`: Tools that are usable and maintained.
- `Archive`: Closed or paused work kept for reference.

## Issue Fields

If using GitHub Projects custom fields, use:

- `Category`: Health, Life, Knowledge, Finance, Inbox, Archive.
- `Status`: Idea, Prototype, Active, Paused, Archived.
- `Priority`: Low, Medium, High.
- `Tool`: Free-text tool name.
